抽象类和接口的区别

抽象类：

  • 可以包含抽象方法（没有具体实现的方法）。
  • 也可以包含具体方法（有具体实现的方法）。

接口：

  • 在Java 8之前，接口只能包含抽象方法。
  • 从Java 8开始，接口可以包含默认方法（default）和静态方法（static），但这些方法必须有具体实现。

先看java中的数据结构，主要是通过结合框架来使用的

集合框架

集合框架提供了一系列用于存储和操作对象的类和接口。

常用接口和类

  • List：有序集合，允许重复元素。

    • ArrayList：基于动态数组实现。

    • LinkedList：基于双向链表实现。

    • Vector：Vector类实现List接口并提供类似ArrayLisst的相同功能，但是同步每个单独的操作。建议使用ArrayList代替Vector。

    • Stack：该类提供堆栈数据结构的功能，继承Vector类。

      本Stack类提供直接执行堆栈的数据结构。但是，建议不要使用它。而是使用ArrayDeque类（实现Deque接口）在Java中实现堆栈数据结构。

  • Set：无序集合，不允许重复元素。

    • HashSet：基于哈希表实现。
    • TreeSet：基于红黑树实现，实现了SortedSet接口，元素有序。不能插入空值，而LinkedHashSet可以插空值。
    • LinkedHashSet：提供了哈希表和链接列表数据结构的功能，类似HashSet的存储，但是在内部为其所有元素维护一个双链表，保持插入顺序，因此存储更大，性能更慢。
    • EnumSet：提供了单个枚举的元素的集合实现。

  • Queue：以先进先出的方式存储和访问元素

    • ArrayDeque：Deque接口的实现方式之一。
    • LinkedList：基于双向链表实现。
    • PriorityQueue：提供堆数据结构的功能。

  • Map：键值对集合，键唯一。

    • HashMap：基于哈希表实现。
    • TreeMap：基于红黑树实现，键有序。
    • EnumMap：基于哈希表实现。
    • LinkedHashMap：基于红黑树实现，键有序。
    • WeakHashMap：基于哈希表实现。

  • Iterator：

为什么使用集合框架？

Java集合框架提供了可以直接使用的各种数据结构和算法。这有两个主要优点：

  • 我们不需要手动编写代码来实现这些数据结构和算法。
  • 随着集合框架的高度优化，我们的代码将更加高效。

此外，集合框架允许我们对特定类型的数据使用特定的数据结构。这里有一些实例，

  • 如果我们希望我们的数据是唯一的，那么我们可以使用集合框架提供的Set接口。
  • 要以键/值对的形式存储数据，可以使用Map接口。
  • ArrayList类提供可调整大小的数组的功能。

Collection 接口包括各种可用于对对象执行不同操作的方法。这些方法在其所有子接口中均可用。

  • add() - 将指定的元素插入到集合中
  • size() - 返回集合的大小
  • remove() - 从集合中删除指定的元素
  • iterator() - 返回一个迭代器以访问集合的元素
  • addAll() - 将指定集合的所有元素添加到集合中
  • removeAll() - 从集合中删除指定集合的所有元素
  • clear() - 删除集合中的所有元素

注意数组和列表的区别：

数组有固定的长度，需要初始化长度大小。而数组列表是动态的

与数组不同，我们不能直接初始化数组列表。 但是，我们可以使用Arrays类的asList()方法来达到相同的效果。

ArrayList的访问方式：使用get(index)来访问，或者使用iterator()顺序访问

for遍历或者for-each遍历

在Java中，我们可以使用toArray()方法将数组列表转换为数组。

也可以使用asList()将数组转换为数组列表。

红黑树

红黑树（Red Black Tree）是一种自平衡二叉查找树。它是在 1972 年由 Rudolf Bayer 发明的，当时被称为平衡二叉 B 树（symmetric binary B-trees）。后来，在 1978 年被 Leo J. Guibas 和 Robert Sedgewick 修改为如今的“红黑树”。

由于其自平衡的特性，保证了最坏情形下在 O(logn) 时间复杂度内完成查找、增加、删除等操作，性能表现稳定。

在 JDK 中，TreeMap、TreeSet 以及 JDK1.8 的 HashMap 底层都用到了红黑树

高性能

Netty

由JBOSS提供的一个java开源框架，现为 Github上的独立项目。Netty提供异步的、事件驱动的网络应用程序框架和工具，用以快速开发高性能、高可靠性的网络服务器和客户端程序。

Dubbo

阿里巴巴公司开源的一个高性能优秀的服务框架，使得应用可通过高性能的 RPC （远程过程调用）

实现服务的输出和输入功能，可以和 Spring框架无缝集成。

核心组件主要包括服务提供者、服务消费者、注册中心、和监控中心。

RocketMQ

阿里巴巴开源的一款分布式消息中间件，其设计目标是处理高吞吐、高可靠、低延迟的消息传递

适用场景：

  • 消息队列
  • 分布式事务
  • 高并发异步处理
  • 日志收集、监控
